## 一、产品概述

### 1.1 产品定位

| 属性 | 说明 |
|------|------|
| **产品名称** | 孕育时光（暂定） |
| **产品类型** | 工具类 + 知识服务类小程序 |
| **核心功能** | 孕期追踪、健康记录、产检管理、知识服务、AI助手 |
| **目标用户** | 备孕/孕期/产后女性（25-35岁） |
| **使用场景** | 日常孕期管理、产检提醒、健康记录 |
| **情感基调** | 温暖、专业、信赖 |

#### 1.2.2 用户分类

| 用户类型 | 占比 | 特征 | 核心功能 |
|---------|------|------|---------|
| **高效型** | 35% | 职场女性，时间宝贵 | 产检提醒、快速记录、AI问答 |
| **记录型** | 30% | 喜欢记录生活 | 体重曲线、胎动记录、报告管理 |
| **学习型** | 25% | 求知欲强 | 每日知识、专家课程、话题讨论 |
| **焦虑型** | 10% | 容易担心 | 异常提示、医生建议、科普内容 |

### 1.3 核心使用场景

| 场景 | 时间 | 行为 | 产品响应 |
|------|------|------|---------|
| **晨起查看** | 7:30-8:00 | 打开小程序看孕周 | 展示今日孕周、胎儿大小、今日知识 |
| **产检当天** | 9:00-11:00 | 记录产检情况 | 产检记录页面，拍照上传报告 |
| **午休浏览** | 12:00-13:00 | 刷孕期知识 | 个性化内容推荐，AI问答入口 |
| **睡前记录** | 22:00-23:00 | 记录体重胎动 | 体重记录弹窗，胎动计数器 |
| **临产焦虑** | 不定时 | 不知道该不该去医院 | 宫缩计时器，AI判断就医时机 |

### 2.2 功能优先级矩阵

| 功能模块 | P0（必须） | P1（重要） | P2（可选） |
|---------|----------|-----------|-----------|
| **首页** | 孕周展示、快捷入口 | 每日知识、产检提醒 | 自定义首页排序 |
| **孕期追踪** | 孕周计算、预产期 | 胎儿发育详情、体重曲线 | 胎儿3D模型 |
| **产检管理** | 日历、提醒、上传报告 | 报告管理、AI解读 | 报告分享、打印 |
| **体重管理** | 体重记录、曲线图 | BMI分析、目标提醒 | 营养建议 |
| **胎动计数** | 计数功能、历史记录 | 异常提醒、统计曲线 | 医生解读 |
| **宫缩计时** | 计时功能、判断提示 | 记录统计 | 医院导航 |
| **待产清单** | 清单模板、勾选完成 | 自定义添加、分享 | 购买链接 |
| **AI助手** | 智能问答 | 报告解读 | 个性化建议 |
| **个人中心** | 孕期信息设置 | 数据导出 | 多账号管理 |

#### 3.1.2 字段说明

| 字段 | 数据来源 | 格式要求 | 说明 |
|------|---------|---------|------|
| 孕周 | 根据末次月经计算 | 第X周 + Y天 | 自动计算 |
| 预产期倒计时 | 根据末次月经计算 | XXX天 | 到预产期天数 |
| 进度条 | 当前孕周/40周 | 百分比 | 孕期进度可视化 |
| 胎儿大小描述 | 预置数据 | 水果/日常物品对比 | 根据孕周匹配 |
| 下次产检 | 用户添加的最近产检 | 日期 + 项目名 | 无则显示"暂无安排" |
| 产检倒计时 | 下次产检日期-当前日期 | X天后/今天/已过期 | 自动计算 |
| 快捷工具数据 | 各功能模块统计 | 数量/状态 | 实时更新 |

#### 3.1.5 权限/隐私

- **脱敏**：首页分享卡片仅包含孕周/对比物/通用建议，不包含预产期具体日期、医院名称、个人备注。
- **通知提示**：右上角“🔔”若用于提醒管理，需要在开启提醒时再触发订阅授权（避免首屏打扰，见 14.1）。

#### 3.2.4 产检时间表（标准版）

| 孕周 | 检查项目 | 说明 |
|------|---------|------|
| 5-8周 | 早孕B超 | 确认宫内孕，可见胎心胎芽 |
| 6-13周 | 建档检查 | 血常规、尿常规、肝肾功能、传染病筛查 |
| 11-14周 | NT检查 | 胎儿颈后透明带厚度，筛查唐氏风险 |
| 15-20周 | 唐氏筛查/无创DNA | 筛查染色体异常 |
| 20-24周 | 大排畸 | 系统超声检查胎儿结构 |
| 24-28周 | 糖耐量检查 | 筛查妊娠期糖尿病 |
| 28-32周 | 小排畸 | 补充排畸检查 |
| 32-34周 | 胎心监护 | 开始胎心监护（NST） |
| 34-36周 | 胎心监护+B超 | 评估胎儿发育、胎位 |
| 36-40周 | 每周产检 | 胎心监护、评估分娩方式 |

#### 3.2.6 权限/隐私

#### 3.13.5 埋点（建议，见 13 章）

| 事件名 | 触发 | 属性（示例） |
|-------|------|--------------|
| me_view | 进入“我的” | - |
| profile_edit_success | 保存孕期信息成功 | status、has_due_date_override |
| settings_push_toggle | 开关提醒 | key、value |
| export_create | 发起导出 | types、format |
| export_download_click | 点击下载导出文件 | export_id |
| favorites_view | 进入我的收藏（P1） | - |

#### 3.13.6 权限/隐私

- **通知权限**：提醒开关开启时，先走订阅/授权引导；用户拒绝后仍可在应用内记录，但提示“可能收不到提醒”（见 4.4.8、14.1）。
- **导出风险提示**：导出前必须二次确认；导出文件名不包含用户昵称/医院名等可识别信息。
- **隐私入口**：关于/隐私页需明确说明收集范围与用途；提供“关闭个性化推荐/关闭AI上下文注入（P0）”入口（对应 `settings.ai.contextEnabled`）。

#### 3.13.7 校验与错误码（见 15 章）

- **孕期信息**：`lastPeriodDate` 必须有效且不晚于今天；`dueDate` 若用户手动覆盖，需要提示“将以此为准计算孕周”（并提供恢复自动计算入口，P1）。
- **导出频控**：频繁导出返回 `E_EXPORT_TOO_FREQUENT`，提示冷却时间。

## 六、API接口设计

### 6.0 通用约定（P0）

#### 6.0.1 通用响应结构

```
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

- **成功**：`code = 0`。
- **失败**：`code != 0`，`message` 给用户可读提示；如需精细处理，使用 `errorCode`（建议与 15 章一致）。

#### 6.0.2 通用错误响应（建议）

```
{
  "code": 10001,
  "message": "参数错误：weight",
  "errorCode": "E_PARAM_INVALID",
  "data": {
    "field": "weight",
    "detail": "must be between 0 and 200"
  }
}
```

#### 6.0.3 幂等与重复提交

| 场景 | 建议 |
|------|------|
| 表单保存（产检/体重/清单新增） | 支持 `Idempotency-Key`（请求头）或客户端生成 `clientRequestId`（请求体） |
| 图片上传 | 单张失败允许重试；多张可并行上传，成功后聚合提交 |

## 九、异常处理设计

### 9.1 网络异常

| 场景 | 处理方式 | 用户提示 |
|------|---------|---------|
| 请求超时 | 自动重试3次 | "网络不稳定，正在重试..." |
| 请求失败 | 显示错误页+重试按钮 | "加载失败，请检查网络后重试" |
| 断网 | 展示缓存数据+离线提示 | "当前处于离线状态，部分数据可能不是最新" |
| 接口500 | 显示错误页 | "服务器异常，请稍后再试" |

### 9.2 业务异常

| 场景 | 处理方式 | 错误码 |
|------|---------|-------|
| 产检日期早于孕周 | 提示"日期早于孕期" | E_CHECKUP_DATE_INVALID |
| 体重记录超出合理范围 | 二次确认弹窗 | E_WEIGHT_UNREASONABLE |
| 上传图片格式不支持 | 提示支持格式 | E_IMAGE_FORMAT_NOT_SUPPORTED |
| 上传图片过大 | 提示压缩或选择其他图片 | E_IMAGE_TOO_LARGE |
| 删除操作 | 二次确认弹窗 | - |
| 重复提交 | 防抖处理 | - |

### 9.3 边界条件

| 条件 | 处理方式 |
|------|---------|
| 孕周计算边界 | 预产期前后2周内显示"随时待产" |
| 体重曲线数据过少 | 少于3条数据时显示引导提示 |
| 产检记录过多 | 分页加载，每页10条 |
| 报告图片过多 | 最多9张，超出提示 |
| 图片上传失败 | 显示失败图片，允许重试单张 |

## 十、交互细节规范

### 10.1 手势操作

| 手势 | 适用场景 | 反馈 |
|------|---------|------|
| **点击** | 按钮、列表项 | 按钮按下态(透明度0.7) |
| **长按** | 删除、更多操作 | 显示操作菜单 |
| **滑动** | 列表左右滑动 | 微信侧滑操作 |
| **下拉** | 刷新列表 | 触发下拉刷新 |
| **上拉** | 加载更多 | 显示loading |

### 10.2 动画规范

| 动画类型 | 时长 | 缓动函数 | 使用场景 |
|---------|------|---------|---------|
| **页面切换** | 300ms | ease-in-out | 页面跳转 |
| **弹窗出现** | 250ms | ease-out | 弹窗展示 |
| **弹窗消失** | 200ms | ease-in | 弹窗关闭 |
| **列表展开** | 200ms | ease-out | 展开/收起 |
| **数字变化** | 150ms | ease-out | 计数动画 |
| **进度更新** | 400ms | ease-in-out | 进度条动画 |

### 10.3 声音反馈

## 十三、埋点与数据口径

### 13.1 设计原则

| 原则 | 说明 |
|------|------|
| **最小可用** | 先覆盖“转化/留存/核心功能使用”，避免一开始埋点过量 |
| **事件命名统一** | 采用 `页面_模块_动作`（snake_case）或 `page.module.action`（二选一，全局统一） |
| **字段可复用** | 公共字段（用户状态/孕周/网络/版本）每条事件自动带上 |
| **隐私优先** | 不上报原始健康指标明细（如具体化验数值、自由文本备注），只上报必要的分段/布尔/枚举 |

| app_version | string | 2.0.0 | 小程序版本 |
| env | enum | prod | dev/stage/prod |
| network | enum | wifi | wifi/4g/5g/none |
| ts | string | 2026-03-27T...Z | 事件时间（UTC） |

## 十四、权限隐私与合规

### 14.1 需要的权限清单（微信侧）

| 权限/能力 | 用途 | 触发时机 | 失败兜底 |
|----------|------|----------|----------|
| 订阅消息/通知 | 产检/每日知识/体重/胎动提醒 | 用户开启提醒或新增产检后 | 提示去授权 + 不影响本地记录功能 |
| 相册/相机 | 上传产检报告图片 | 点击“上传报告” | 允许仅填摘要/稍后上传 |
| 文件下载/打开 | 数据导出下载 | 导出 ready 后 | 提示复制下载链接（H5）或稍后重试（可选） |

### 14.2 数据分级与脱敏策略

| 数据类型 | 示例 | 存储/传输 | 埋点上报 |
|---------|------|-----------|----------|
| **强敏感（健康明细）** | 化验数值、B超指标、自由文本备注 | 必须走 HTTPS；服务端加密存储（建议） | **禁止**原文上报，仅可上报分段/枚举 |
| **中敏感（健康汇总）** | “血糖正常/异常”、AI解读结论标签 | 可存；展示需免责声明 | 可上报枚举（normal/abnormal/unknown） |
| **弱敏感（行为数据）** | 页面PV、按钮点击 | 可存 | 可上报 |

### 14.3 AI 内容安全与医疗免责声明（P0）

- **统一免责声明**：AI 输出与健康内容页底部固定展示“仅供参考，不替代医生诊疗”。
- **高风险触发**：检测到“出血/破水/胎动明显减少/剧烈腹痛”等关键词时，优先输出就医建议与急救指引，不输出诊断结论。
- **上下文可关闭**：`settings.ai.contextEnabled=false` 时，AI 不注入用户孕周/记录数据。

## 十五、输入校验与错误码

### 15.1 校验原则

| 原则 | 说明 |
|------|------|
| **前后端一致** | 前端做即时校验 + 友好提示；后端做最终校验并返回错误码 |
| **不阻断记录** | 对“可疑数据”采用二次确认，而非直接禁止（如体重异常值） |
| **可配置阈值** | 胎动/宫缩提示阈值由服务端配置，避免客户端硬编码 |

### 15.2 通用错误码（建议）

| 错误码 | 含义 | 典型处理 |
|-------|------|----------|
| E_UNAUTHORIZED | 未登录/令牌失效 | 引导重新授权登录 |
| E_FORBIDDEN | 无权限 | 提示联系支持或检查权限 |
| E_RATE_LIMITED | 频控 | 提示稍后再试（带冷却时间） |
| E_PARAM_INVALID | 参数校验失败 | 聚焦到具体字段提示 |
| E_NOT_FOUND | 资源不存在 | 返回上一页 + Toast |
| E_CONFLICT | 冲突（重复提交/版本冲突） | 刷新数据后重试 |
| E_NETWORK | 网络异常 | 展示离线/重试 |

### 15.3 业务错误码补充（扩展 9.2）

| 错误码 | 场景 | 说明 |
|-------|------|------|
| E_FM_SESSION_RUNNING | 胎动 session 未结束又创建 | 引导恢复/先结束当前 session |
| E_CONTRACTION_TIME_INVALID | 宫缩结束早于开始 | 前端阻止 + 后端校验 |
| E_CHECKLIST_ITEM_TOO_LONG | 清单自定义项过长 | 限制 title 1-20 字、note 0-50 字（可配） |
| E_ARTICLE_CONTENT_UNAVAILABLE | 文章内容不可用 | 展示占位 + 重试 |
| E_EXPORT_TOO_FREQUENT | 导出频繁 | 限制 1 小时 1 次（可配） |
| E_AI_SAFETY_BLOCKED | AI 安全拦截 | 返回安全提示文案 + 推荐内容 |

### 15.4 关键字段校验表（P0）

| 模块 | 字段 | 规则 |
|------|------|------|
| 用户 | lastPeriodDate | 必须为日期；不得晚于今天；不得早于合理范围（如 2 年前） |
| 产检 | checkupDate | 不得早于 lastPeriodDate（孕期）过多；允许用户手动记录历史（需提示） |
| 体重 | weight | 30-200kg 超出二次确认；精度 0.1 |
| 胎动 | count | 0-999；结束时若为 0 提示“是否误触/不保存” |
| 宫缩 | startedAt/endedAt | endedAt >= startedAt；duration 5s-10min 超出提示 |
| 清单 | title/note | title 1-20字；note 0-50字；去除特殊控制字符 |
| AI | question | 1-500字；过滤明显隐私（手机号/身份证）并提示用户勿发送 |
